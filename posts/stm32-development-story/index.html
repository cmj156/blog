项目简介#

这是一个非接触式手势控制自平衡平台——用手在红外传感器上方移动，平台就会跟着倾斜，同时保持上面的小球不掉落。

核心技术栈：

  • 主控：STM32F103C8T6（72MHz Cortex-M3）
  • 姿态传感：MPU6050（加速度计 + 陀螺仪）
  • 执行器：2路SG90舵机（俯仰 + 横滚双轴）
  • 交互：5路红外传感器检测手部位置 + OLED显示

开发历程#

第一阶段：硬件搭建#

最早的需求很简单——做一个能自平衡的平台。选了 STM32F103C8T6，因为便宜、资料多、HAL库成熟。

硬件连接：

  • MPU6050 接 I2C2（PB10/PB11）
  • OLED 接 I2C1（PB6/PB7）
  • 5路红外传感器接 PA1-PA5（外部中断）
  • 2路舵机接 TIM1 CH1/CH2（PWM输出）
  • UART1 用于调试和在线调参

第二阶段：传感器驱动#

MPU6050 驱动是最先完成的模块。通过 I2C 读取原始加速度和角速度数据，配置量程：

  • 加速度计：±2g
  • 陀螺仪：±2000°/s

最初直接用加速度计算角度，发现噪声很大，平台抖动严重。这就是为什么需要卡尔曼滤波。

第三阶段：卡尔曼滤波#

这是整个项目最关键的算法之一。用一个2状态卡尔曼滤波器融合加速度计和陀螺仪的数据：

状态向量 = [角度, 偏置]
观测向量 = [加速度计角度]

参数选择：

  • Q_angle = 0.001（过程噪声）
  • R_measure = 0.03（测量噪声）

效果：角度输出平滑且响应快，200Hz控制循环下运行稳定。

第四阶段：PID 控制器#

PID 是控制舵机的核心。但不是简单的 PID，加入了很多工程优化：

特性说明
微分项作用于测量值避免设定值突变导致的微分冲击
微分低通滤波抑制高频噪声
设定值加权平衡响应速度和稳定性
死区设置2.0°死区抑制机械间隙导致的振荡
条件抗积分饱和防止积分项过大

俯仰轴目标：保持水平（0°） 横滚轴目标：跟随红外传感器检测的手部位置（-8° ~ +8°）

第五阶段：红外传感器#

5路红外传感器用外部中断触发，带20ms硬件去抖。

最大的问题是信号丢失处理——手离开传感器范围时，不能让平台突然跳到极端位置。解决方案是加一个斜坡函数：

// 信号丢失时，目标角度以0.5°/step平滑回归零位
if (signal_lost) {
    target = target * 0.9;  // 渐近归零
}

第六阶段：显示与调试#

OLED 显示做了帧率控制——每20帧（100ms）刷新一次，或者状态变化时立即刷新。这样避免 I2C 总线冲突。

UART 调参协议设计为：

P:2.5,0.01,0.8,0.5

格式：P:Kp,Ki,Kd,Deadband，串口发送即可实时修改参数，无需重新烧录。

第七阶段：上电校准#

开机时自动执行水平校准——用比例控制驱动舵机，直到平台倾斜角小于0.5°。整个过程大约3秒完成。

最终成果#

指标数值

编译警告0

项目代码结构清晰，中文注释完整，零编译错误/警告。

经验总结#

  1. 传感器融合比单传感器可靠得多 — 卡尔曼滤波是嵌入式姿态估计的标配
  2. PID不是调好就行 — 抗积分饱和、微分滤波、死区这些"小细节"决定了实际效果
  3. 异常处理很重要 — 信号丢失、机械间隙、启动校准，这些才是工程化的关键
  4. 串口调参效率极高 — 比改代码重新烧录快10倍
